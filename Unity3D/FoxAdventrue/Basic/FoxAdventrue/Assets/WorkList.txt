4. 상호작용하기
-사다리만들기: 사다리에 도착하면 위로 키를 눌러서 사다리를 올라갈수 있는 것. -> 사다리에 충돌체크되면 사다리상태로 변경하고, 사다리 상태일때 중력을 제거하고 업/다운키를 이용하여 조작가능 -
-똑똑한AI?: 똑똑하다? 정확하다. 다양한 상황에 대응 할수 있어야한다.-> 플레이어를 발견하면 쫓아온다.
-플레이어를 (발견)하면 (쫓아온다:플레이어쪽으로 (이동)한다).
-발견: 시야안에 들어왔다. -> 플레이어를 타겟으로 변경한다. -> 시야에서 사라지면, 타겟을 돌아가는 위치로 변경하고, 돌아가기 상태로 만든다.
-이동(추적): 이동하는 방향이 플레이어가 있는 위치에따라 실시간으로 변경된다.
-복귀: 플레이어가 시야에서 사라지면 원래 위치로 돌아간다. -> 타겟을 리스폰위치로 변경한다.
-정찰: (시작하는 위치)에서 (정찰위치)로 (이동)하고, 정찰위치에서 시작하는 위치로 다시 돌아온다(목적지에 도착 후 이동).
-부활: (독수리:게임오브젝트)가 죽었을때 (리스폰위치:트랜스폼)에서 (독수리가 살아난다.) -> 없는 독수리가 복사해서 만든다. -> 게임오브젝트가 비활성화 될때 게임오브젝트를 복제하여 활성화 시킨다.
  ->대상이 존재하지않은 상태에서 무엇인가 진행이되야하므로, 삭제된대상에 수행하기 어렵다. 그러므로, 부활할 위치에 제어하는 스크립트를 추가하여 3자입장에서 프로그래밍 해야한다.
-개구리: 정찰하고 플레이어를 발견하면 쫓아옴. -> 개구리처럼 이동해야함. -> 개구리는 점프하면서 이동한다. -> 매뚜기?가 될수있음. 개구리와 매뚜기의 차이에 대서 생각해보기 -> 일정시간동안 대기후 점프를 다시하도록 만든다.

-아이템을 사용했으나 UGUI에서 사용한 아이템이 사라지지않는 현상
ㄴ아이템을 실제 데이터부분은 수정되었으나 gui를 실제 데이터에 맞게 업데이트하지않음.
-아이템을 버튼과 설명에 이미지와 실제 아이콘색상이 다른부분 맞게 변경된다

ㄴ아이템인벤토리가 열릴/아이템사용 한경우 첫번째 인벤토리에 아이템정보를 표시하고, 없는 경우 클리어하며, gui초기화시에 내용을 모두 클리어하도록 수정.
